A Performance Art?

What does that mean?

It means…

CHANGE = "The Only Constant"

It means your code is…

“A momentary flash of brilliance, doomed to be overtaken by the next wave, or maybe even by its own sequel. Eaten alive by its successors. And time…”

Kai Krause · http://www.edge.org/q2008/q08_16.html#krause

It means

The Art of Programming

is guiding your code towards a graceful demise.

☠

How do we do that?

First, there are two conflicting principles
you must understand:

“All bugs come from code.
Write less of that.”

“I didn’t have time to write a short letter,
so I wrote a long one instead.”

Mark Twain

Which leads to…

“This code is shit.
I haven’t had time to refactor it.”

All of us, at some point.

It’s easy to write a lot of code,
and we have deadlines to meet.

So we end up with code debt.

Process

In all creative work there are two fundamental actions:

  1. Breaking Things Down
  2. Building Things Up

During each project you’ll oscillate between them both.

Breaking Things Down ⬇ ⬆ Building Things Up
Gathering Specs ⬇ ⬆ Writing Tests
Designing an Interface ⬇ ⬆ Building a Prototype
Code Review ⬇ ⬆ Deploy

☯

So don’t over-complicate things.

We have shit to do.

Code Debt

There’s no easy answer.

  • Don’t write code debt in the first place.
  • Make the time to pay it.
  • Write it off.

Principles

for the Developer

Honesty

Don’t write clever code.

Walk the path of least bullshit.

Detachment

Don’t marry your code.

The probability of someone working on your code that isn’t you is very high.

Let it go.

There will always be more code to write.

Humility

Don’t assume you have all the answers.

Test your assumptions.

Test your code.

Integrity

Leave the code you work with better than you found it.

You’ll be doing someone a favor, even if it’s future you.
